1. Executive Summary

Antigravity Mobile began as a bridge utility for the Antigravity IDE and has evolved into a standalone product that enables developers to interact with AI coding agents from their mobile device — without being tethered to a desktop. By routing all communication through MemFlow's local SQLite store rather than fragile OS-level automation (AppleScript, Accessibility APIs), the product achieves IDE-agnostic, OS-stable operation.

The core thesis: MemFlow is the message bus. Any IDE that exposes a MemFlow MCP server connection gains full mobile coding enablement — inbox delivery, agent response relay, project context, and session continuity — for free.

2. Problem Statement

Developer Pain

  • AI coding agents run on desktop only
  • No way to send instructions from mobile
  • Agent responses not visible off-desk
  • Long-running tasks require physical presence
  • Context lost between mobile and desktop sessions

Infrastructure Pain (v0)

  • AppleScript brittle — breaks on macOS updates
  • Accessibility permission prompts block automation
  • CDP-only injection is one-directional
  • No response path from agent back to mobile
  • HTTP-only exposure caused "Not Secure" warnings
  • Server restarts required manual intervention

4. Key Design Decisions

MemFlow as Local-Only Bus

All ag_bridge/* namespaces use scope: workspace, writing only to the local SQLite instance. Mobile session data never contaminates shared or global MemFlow namespaces, preserving data sovereignty across projects and teams.

No AppleScript

Previous versions depended on AppleScript + Accessibility APIs to simulate keypresses. This broke on macOS updates and required explicit system permissions. The MemFlow MCP path is OS-stable, headless-compatible, and works identically on macOS, Linux, and Windows.

Exponential Backoff Reconnect

The mobile WebSocket client retries at 1s → 2s → 4s → 8s → 16s intervals with stale socket cleanup. A background health poll (10s) detects server-down states independently of the WS connection state machine.

launchd Auto-Heal

KeepAlive: true restarts the bridge on any exit. The /admin/restart endpoint exits with code 1 (not 0) to trigger launchd's restart, eliminating race conditions from previous bash-script spawning approach.

5. MCP Plugin API (v1)

7. Product Roadmap

Sprint 1 — Foundation Hardening Complete

Current prod baseline. All items shipped and tagged v1.0.0-prod.

  • HTTPS persistence via Tailscale serve + dedicated launchd plist
  • HTTP → HTTPS redirect middleware (X-Forwarded-Proto detection)
  • Exponential backoff WebSocket reconnect + connection status banner
  • launchd KeepAlive hardening + graceful restart via exit(1)
  • Active projects deduplication, sort, and live CDP activity stamping
  • MemFlow MCP tools: mobile_read_inbox + mobile_respond
  • Bidirectional message loop: mobile → MemFlow inbox → agent → outbox → mobile

  • Define IDEConnector interface: getTargets(), poke(), watchOutput()

8. Security Model

Transport Security

  • All traffic over HTTPS/WSS via Tailscale (Let's Encrypt cert)
  • Direct HTTP access auto-redirected to HTTPS
  • Tailscale ACLs limit access to authenticated Tailnet devices
  • No public internet exposure by default

Auth & Isolation

  • Bearer token authentication on all non-localhost endpoints
  • Localhost (MCP server) bypasses auth for agent-to-bridge calls
  • MemFlow data: scope: workspace — local SQLite only
  • ag_bridge/* namespaces never sync to shared stores
